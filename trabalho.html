Projeto e Implementação

Desenvolvimento

É o processo de conversão de uma especificação de sistema em um sistema executável. Esse estágio sempre envolve processos de projeto e programação de software, mas, se uma abordagem evolucionária de desenvolvimento for utilizada, ele poderá também envolver a aperfeiçoamento da especificação de software.

Um projeto de software é uma descrição de estrutura de software a ser implementada dos dados que são parte do sistema, das interfaces entre os componentes do sistema e, algumas vezes, dos algoritmos utilizados. Os projetistas não conseguem que um projeto seja concluido imediatamente, mas desenvolvem o projeto interativamente por meio de uma série de diferentes versões. O processo de projeto envolve acrescentar formas e detalhes, à medida que o projeto é desenvolvido, com 'retornos' constantes, a fim de corrigir projetos anteriores.

Uma especificação para o próximo estágio é a saída de cada atividade do projeto Essa especificação pode ser uma especificação de como parte do sistema é implementada A medida que o processo de projeto continua, essas especificações se tornam mais detalhadas: Os resultados de processo são especificações precisas de algoritmos e estruturas de dados a serem implementadas.

As atividades de processo de projeto são:
  • Projeto de arquitetura - Definir a estrutura geral do sistema, incluindo os componentes principais, suas interações e a organização geral do software.
  • Especificação abstrata - Desenvolver uma descrição de alto nível do sistema, focando nas funcionalidades e comportamentos principais, sem entrar em detalhes de implementação.
  • Projeto de interface - Projetar as interfaces entre os diferentes componentes do sistema, determinando como eles se comunicam e trocam informações.
  • Projeto de componentes - Identificar e projetar os componentes individuais do sistema, definindo suas funcionalidades específicas e como eles se encaixam na arquitetura geral.
  • Projeto de estrutura de dados - Projetar as estruturas de dados necessárias para armazenar e manipular as informações dentro do sistema, considerando eficiência e organização.
  • Projeto de algoritimos - Desenvolver os algoritmos e métodos necessários para realizar as operações e funcionalidades requeridas pelo sistema, garantindo a corretude e eficiência das soluções.

Conclusão

O processo de desenvolvimento de software envolve a conversão de uma especificação de sistema em um sistema executável. Esse estágio requer processos de projeto e programação de software, e se uma abordagem evolucionária for adotada, também pode incluir melhorias na especificação. Um projeto de software descreve a estrutura a ser implementada, incluindo dados, interfaces e possivelmente algoritmos. O projeto é desenvolvido de forma iterativa, com várias versões que são refinadas ao longo do tempo, com base em feedback constante. A cada etapa do projeto, uma especificação é produzida, que pode detalhar a implementação de uma parte específica do sistema. À medida que o processo de projeto avança, essas especificações se tornam mais detalhadas e resultam em especificações precisas de algoritmos e estruturas de dados a serem implementados. As atividades do processo de projeto incluem o projeto de arquitetura, especificação abstrata, projeto de interface, projeto de componentes, projeto de estrutura de dados e projeto de algoritmos.
